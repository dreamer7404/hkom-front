DEXT5 Editor :: UseHtmlCorrection

XML : <use_html_correction>.....</use_html_correction>, Config : config.UseHtmlCorrection
XML : <use_html_correction limit_length="....."></use_html_correction>,
Config : config.HtmlCorrectionLimitLength
XML : <use_html_correction remove_incorrect_attribute="....."></use_html_correction>,
Config : config.RemoveIncorrectAttribute
XML : <use_html_correction replace_space="....."></use_html_correction>, Config : config.ReplaceSpace
XML : <use_html_correction skip_tag="....."></use_html_correction>, Config : config.SkipTagInParser

Config 설정이 우선 적용됩니다.
set API 사용시 htmlparser로직(사용)의 실행 여부 및 parser의 동작을 설정합니다.

remarks

use_html_correction  기본값은 "1" 이며 set API 사용시 htmlparser로직(사용) 실행, "0" 으로 설정 시 실행하지 않습니다.
limit_length  기본값은 450000이며 html의 길이 제한을 설정합니다. (메모리 부족 오류 시 브라우저가 닫혀버리는 현상을 방지가능합니다.)
remove_incorrect_attribute  기본값은 "0" 이며 사용안함, "1" 로 설정시 올바르지 않은 속성값을 pareser에서 삭제하도록 설정합니다.
replace_space  기본값은 "0" 이며 사용안함, "1" 로 설정시 parser에서 공백 2개를 &nbsp; + 공백 으로 바꾸도록 설정합니다.
skip_tag  parser에서 html 보정 시 skip할 tag를 설정합니다.

sample code

set API 사용시 htmlparser로직(사용)의 실행 여부 및 parser의 동작을 설정합니다.

XML

.....
<use_html_correction limit_length=500000 remove_incorrect_attribute="1" replace_space="0" skip_tag="h1">1</use_html_correction>
.....

Config

<script type="text/javascript" src="dext5editor/js/dext5editor.js"></script>	
    
<!-- ..... 생략 ..... -->
    
<div style="width:900px;height:550px">    
    <script type="text/javascript">

        DEXT5.config.UseHtmlCorrection = "1";
        DEXT5.config.HtmlCorrectionLimitLength = 500000;
        DEXT5.config.RemoveIncorrectAttribute = "1";
        DEXT5.config.ReplaceSpace = "0";
        DEXT5.config.SkipTagInParser = "h1";

    </script>       
</div>
