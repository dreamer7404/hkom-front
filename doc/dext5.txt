Spring Boot에서 덱스트5 솔루션 핸들러 서블릿 구성
덱스트5 솔루션은 파일 업로드를 위한 서버 모듈을 제공해드리고 있습니다.(에디터, 업로드 모두 제공하고 있습니다.)

이 서버모듈을 spring boot 환경에서 구성하실 경우 핸들러 파일을 서블릿으로 구성 후 사용하실 수 있습니다.

이 때 Spring Boot 의 Multipart 설정이 true 로 되어있는 경우 Multipart가 미리 읽히는 문제가 발생할 수 있는데요,

Java 에서는 Multipart 를 한번 읽게되면 다시 읽을 수 없기 때문에 업로드 시 에러가 발생할 수 있습니다.

이러한 경우에 대해 예외처리를 하는 샘플 소스 보내드리니 참고하여 적용해주시면 됩니다.

(아래 내용은 덱스트5 에디터 기준으로 작성된 내용입니다.)

 

[Spring Boot 연동 샘플]

제품폴더(.../dext5editor/) 하위 handler 폴더 내 upload_handler.jsp 파일을 서블릿으로 구성 후 Handler URL을 해당 서블릿 URL로 변경하여 적용하시면 됩니다.

 

서블릿 구성 시 주의사항

1.     Get, Post 모두 허용

2.     Return Type은 void

3.     upload_handler.jsp 에 포함되어 있는 서블릿 구성 시 참고 사항대로 적용

 

다운로드

l  @SpringBootApplication이 선언된 (프로젝트명)Application.java에 customMultipartResolver, multipartFilter 등록

(첨부된 파일 중 TestApplication.java에 소스코드 존재합니다.)

 

...(생략)…

 

è  @Bean

public CustomMultipartResolver customMultipartResolver() {

CustomMultipartResolver multipart = new CustomMultipartResolver();

multipart.setMaxUploadSize(3 * 1024 * 1024);

return multipart;

}

 

è  @Bean

@Order(0)

public MultipartFilter multipartFilter() {

MultipartFilter multipartFilter = new MultipartFilter();

multipartFilter.setMultipartResolverBeanName("customMultipartResolver");

return multipartFilter;

}

(첨부된 파일 중 CustomMultipartResolver.java 파일을 적용하시면 됩니다.)

l  Spring Boot에 포함되어 있지 않은 commons-fileupload-1.4.jar 적용

(첨부된 파일 중 commons-fileupload-1.4.jar 파일을 WEB-INF/lib에 적용하시면 됩니다.)

 

l  서블릿 URL로 Handler URL을 수정 (아래 내용 참고)

 

DEXT5 Editor :: HandlerUrl

XML : <handler_url>.....</handler_url>, Config : config.HandlerUrl

Config 설정이 우선 적용됩니다.
웹에서 RFC 1867 표준에 따르는 POST 방식의 파일 전송을 받아주는 페이지 URL입니다.
이미지 파일 또는 Flash, media 파일 등 서버에 파일이 업로드 되지 않을 경우 이 값의 설정을 확인하세요.

 

remarks

기본값은 "" 입니다.
dext5editor/handler/ 아래의 각 언어에 맞는 페이지를 호출합니다.
상대 및 절대 경로 모두를 지원합니다.

지정된 URL의 동작을 확인 하려면 handler_url URL에
"?dext=test"를 붙여서 테스트 후 페이지에 "Hi, DEXT5 Editor !!!! test" 가 표시되면 정상적으로 설정된 경우입니다.

각 개발 언어에 맞는 값으로 변경 설정하세요.
