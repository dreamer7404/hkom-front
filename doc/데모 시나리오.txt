본시스템을 개발을 맡고있는 ccmedia의 안경수 입니다.

이 프로젝트는 현대/기아의 데이타 통합과 사용자 인터페이스의 개선을 목표로 하고있고, 
8월말에 개발 완성에 앞서서, 파이럿버전의 프로그램을 선행적으로 보여드릴려고 합니다.

데이터 통합관련 배치프로세스는 현재 진행중으로 아직 통합된데이터는 나오지 않고있습니다.
사용자 인터페이스는 현재 화면을 크게 변화하는것은 혼란을 줄수 있기때문에 불편한 요소를 개선하는것에 중점을 두었습니다.

현재의 파일럿프로그램은 진행중인 화면이라 테스트용 데이터를 사용중인것을 감안해주셨으면 합니댜.
그래서 화면 디자인과 사용자 인터페이스에 초점을 두시고 봐주셨으면 합니다.

1. 로그인
	- 우선 로그인화면부터 보시겠습니다.
	- 로그인은 현대자동차, 기아자동차 2개로 나누어져 있고, 각각 협력업체로 로그인할수있게 인터넷주소를 설정할 예정입니다.
	- 비밀번호 변경주기는 6개월이며, 앞으로 미변경시에는 로그인이 안될예정.


2. 전체화면 구성
	전체화면을 보시면 위쪽에 메뉴헤더부분이 있고, 그 아래에 조회영역이 있습니다.
	그리고 그아레에 그리드로 데이타를 나타내게 구성하였습니다.

	(1) 헤더
		- 메뉴는 권한그룹별로 설정한대로만 나옵니다.
		- 마우스 오버시 메뉴나옴.
		
		- 재고관리: "게시판"이 추가.
		- 제작준비: O/M제작진행, 발주리스트, O/M발주 => O/M발주 병합 ??
		- 선적현황 추가
		- 발간현황: "세원의뢰조회" 삭제 ??
		- 운영관리: "안전제고관리" 삭제 ??
		- 시스템관리: 이메일관리 추가.


		- 시스템관리자는 현대/기아를 번걸아가며 볼수잇습니다.

	(2) 조회영역
		- 조회영역은 필요없거나 그리드 데이타부분을 좀 더 크게 보고싶을때에는 감출수있습니다.
		- 선택버튼을 눌러도 되지만, 선택할때마다 
		- 모든화면의 조회영역의 선택값은 공기화되어 있어서, 한번 선택한 값은 다른화면으로 이동해도 다시 선택할 필요가 없이 동기화되어있습니다.
		- 그러나, 팝업화면은 별도의 로직이기 때문에 따로 선탹하여 사용하셔야 합니다.
		- My입력부분은 목록에서 선택하는 방식으로 변경.

	(3) 데이터 영역
		- 기존의 팝업창을 탭으로 구분하여, 시인성과 조회조건들을 동기화하여 사용편의성을 좋게.
		- 대부분의 데이터는 그리드콤포넌트를 사용해서 구현했습니다.
		- 페이징기능의 개선으로 화면새로고침없이, 데이터만 변경되어 화면을 다시 내려야하는 불편함을 없엤.
		- 모든 컬럼이 자리이동 가능하며, 정렬도 가능합니다.
		- 필터기능이 필요한 화면은 통합필터기능을 사용했고, 컬럼별로 필터링 필요한부분은 컬럼별로 구현했습니다.
